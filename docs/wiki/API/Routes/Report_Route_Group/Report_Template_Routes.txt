====== Report Template Routes ======

===== Overview =====

API endpoints for managing report templates. Templates define which metrics should be included when creating a report for a specific timeframe kind.

Users can have multiple templates per kind (e.g., "Weekday Daily", "Weekend Daily") and mark one as default.

All endpoints require authentication via JWT Bearer token.


===== Endpoints =====

==== List Templates ====

**GET /api/report-templates**

Returns all templates for the authenticated user.

| Parameter | Type | Location | Required | Description |
|-----------|------|----------|----------|-------------|
| kind | string | query | no | Filter by timeframe kind: day, week, month, season, year |

**Response:** Array of template objects


==== Create Template ====

**POST /api/report-templates**

Creates a new report template.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| name | string | yes | Template name (max 100 chars) |
| timeframe_kind | string | yes | day, week, month, season, year |
| is_default | boolean | no | Set as default for this kind (default false) |
| metric_type_ids | array | no | Array of metric type IDs to include initially |

**Response:** Created template object (201)

**Example Request:**
	name: "Daily Standard"
	timeframe_kind: "day"
	is_default: true
	metric_type_ids: [1, 2, 3, 4]


==== Get Template ====

**GET /api/report-templates/<template_id>**

Returns a specific template by ID.

**Response:** Template object, or 404 if not found


==== Update Template ====

**PUT /api/report-templates/<template_id>**

Updates a template's properties.

| Field | Type | Description |
|-------|------|-------------|
| name | string | Template name (cannot be empty) |
| timeframe_kind | string | day, week, month, season, year |
| is_default | boolean | Set as default for this kind |

**Response:** Updated template object, or 404 if not found


==== Delete Template ====

**DELETE /api/report-templates/<template_id>**

Permanently deletes a template. Does NOT affect existing reports created from this template.

**Response:** 204 No Content, or 404 if not found


==== Set Default Template ====

**POST /api/report-templates/<template_id>/set-default**

Sets a template as the default for its timeframe kind. Only one template per (user, timeframe_kind) can be default.

**Response:** Updated template object, or 404 if not found


==== Get Template Metrics ====

**GET /api/report-templates/<template_id>/metrics**

Returns all metrics in a template, ordered by sort_order.

**Response:** Array of template metric objects, or 404 if template not found


==== Add Metric to Template ====

**POST /api/report-templates/<template_id>/metrics**

Adds a metric to a template.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| metric_type_id | integer | yes | ID of the metric type to add |
| sort_order | integer | no | Position in the template (defaults to end) |

**Response:** Created template metric object (201), or 404 if template not found

**Errors:**
* 400 if metric_type_id not found
* 400 if metric already in template


==== Reorder Template Metrics ====

**PUT /api/report-templates/<template_id>/metrics**

Reorders metrics in a template.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| metric_type_ids | array | yes | Array of metric type IDs in desired order |

**Example:**
	metric_type_ids: [3, 1, 4, 2]

**Response:** Array of updated template metric objects, or 404 if template not found


==== Remove Metric from Template ====

**DELETE /api/report-templates/<template_id>/metrics/<metric_type_id>**

Removes a metric from a template. Does NOT delete the MetricType.

**Response:** 204 No Content, or 404 if template not found


===== Template Object =====

| Field | Type | Description |
|-------|------|-------------|
| id | integer | Primary key |
| user_id | integer | Owner |
| name | string | Template name |
| timeframe_kind | string | day, week, month, season, year |
| is_default | boolean | Whether this is the default for its kind |
| metrics | array | Embedded template metric objects |
| created_at | datetime | When created |
| updated_at | datetime | When last modified |


===== Template Metric Object =====

| Field | Type | Description |
|-------|------|-------------|
| id | integer | Primary key |
| template_id | integer | Parent template |
| metric_type_id | integer | Metric type reference |
| sort_order | integer | Position in template |
| metric_type | object | Embedded metric type object |
| created_at | datetime | When created |
| updated_at | datetime | When last modified |


===== Workflows =====

==== Setting Up Daily Tracking ====

1. Create metric types for what you want to track:
	POST /api/metric-types with name="Sleep Hours", value_type="integer", unit="hours"
	POST /api/metric-types with name="Work RPE", value_type="rating", min_value=1, max_value=10

2. Create a daily template with those metrics:
	POST /api/report-templates with name="Daily Standard", timeframe_kind="day", is_default=true, metric_type_ids=[1, 2]

3. When accessing a day's report, the template is applied automatically:
	POST /api/reports/for-date with kind="day", date="2026-01-12", timezone="America/Chicago"

The report is created with empty metric value slots ready to fill in.


==== Changing Your Template ====

When your tracking needs change:

1. Archive old metrics you no longer want:
	POST /api/metric-types/5/archive

2. Create new metrics:
	POST /api/metric-types with name="New Metric", value_type="boolean"

3. Update your template:
	DELETE /api/report-templates/1/metrics/5
	POST /api/report-templates/1/metrics with metric_type_id=6

Existing reports keep their old metrics. New reports get the updated template.
