====== Tag Routes ======

===== Overview =====

API endpoints for managing tags. Tags provide flexible, low-level categorization across entities. They're meant to be numerous and used liberally for organization, filtering, and grouping related items.

Examples: "work", "home", "urgent", "health", "learning", "errands"

All endpoints require authentication via JWT Bearer token.


===== Tag Endpoints =====

==== List Tags ====

**GET /api/tags**

Returns all tags for the authenticated user.

**Response:** Array of tag objects (200)


==== Create Tag ====

**POST /api/tags**

Creates a new tag.

| Field | Type   | Required | Description                                             |
|-------|--------|----------|---------------------------------------------------------|
| name  | string | yes      | Tag label (max 50 chars, cannot be empty)               |
| color | string | no       | Hex color code (max 10 chars). Default "#6c757d" (gray) |

**Response:** Created tag object (201)

**Errors:** 400 if name is missing or empty


==== Get Tag ====

**GET /api/tags/<tag_id>**

Returns a specific tag by ID.

**Response:** Tag object (200), or 404 if not found


==== Update Tag ====

**PATCH /api/tags/<tag_id>**

Updates a tag's properties.

| Field | Type   | Description                                           |
|-------|--------|-------------------------------------------------------|
| name  | string | Tag label (max 50 chars, cannot be empty if provided) |
| color | string | Hex color code (max 10 chars)                         |

**Response:** Updated tag object (200), or 404 if not found

**Errors:** 400 if name is set to empty


==== Delete Tag ====

**DELETE /api/tags/<tag_id>**

Permanently deletes a tag and removes it from all attached entities via cascade.

**Response:** 204 No Content, or 404 if not found


===== Association Endpoints =====

==== Attach Tag to Entity ====

**POST /api/tags/attach**

Attaches a tag to an entity.

| Field       | Type    | Required | Description                         |
|-------------|---------|----------|-------------------------------------|
| tag_id      | integer | yes      | ID of the tag to attach             |
| target_type | string  | yes      | Entity type (see taggable entities) |
| target_id   | integer | yes      | ID of the target entity             |

**Business Logic:**
* Idempotent: attaching the same tag twice has no effect
* Entity must support tags (have TaggableMixin)

**Response:** { "success": true } (200)

**Errors:**
* 400 if required fields missing or if tag attachment fails
* 404 if tag or target entity not found
* 422 if target_type is invalid (returns error object with "code": "INVALID_TARGET_TYPE")


==== Detach Tag from Entity ====

**POST /api/tags/detach**

Removes a tag from an entity.

| Field       | Type    | Required | Description                         |
|-------------|---------|----------|-------------------------------------|
| tag_id      | integer | yes      | ID of the tag to detach             |
| target_type | string  | yes      | Entity type (see taggable entities) |
| target_id   | integer | yes      | ID of the target entity             |

**Business Logic:**
* Idempotent: detaching a non-attached tag has no effect

**Response:** { "success": true } (200)

**Errors:**
* 400 if required fields missing or if tag detachment fails
* 404 if tag or target entity not found
* 422 if target_type is invalid (returns error object with "code": "INVALID_TARGET_TYPE")


===== Taggable Entities =====

The following entity types are supported for tag attachment/detachment:

| Entity Type | Description                    |
|-------------|--------------------------------|
| task        | A task (standalone or subtask) |
| project     | A project                      |
| routine     | A routine template             |
| session     | A specific session occurrence  |
| calendar    | A calendar container           |
| report      | A report                       |


===== Tag Object =====

| Field      | Type       | Description        |
|------------|------------|--------------------|
| id         | integer    | Primary key        |
| user_id    | integer    | Owner              |
| name       | string(50) | Tag label          |
| color      | string(10) | Hex color code     |
| created_at | datetime   | When created       |
| updated_at | datetime   | When last modified |
