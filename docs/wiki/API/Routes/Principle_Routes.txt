====== Principle Routes ======

===== Overview =====

API endpoints for managing principles. Principles represent high-level values, aspirations, or "why" statements. They're meant to be few in number (5-10 total) and represent core motivations that can be linked to actions.

The idea: define what kind of person you want to be, then connect tasks/projects/routines to those ideals.

Examples: "Be more reliable", "Prioritize health", "Continuous learning", "Financial independence"

All endpoints require authentication via JWT Bearer token.


===== Principle Endpoints =====

==== List Principles ====

**GET /api/principles**

Returns all principles for the authenticated user.

**Response:** Array of principle objects


==== Create Principle ====

**POST /api/principles**

Creates a new principle.

| Field       | Type   | Required | Description                                  |
|-------------|--------|----------|----------------------------------------------|
| title       | string | yes      | Principle statement (max 100 chars)          |
| description | text   | no       | Elaboration on what this principle means     |
| color       | string | no       | Hex color code. Default "#ffd700" (gold)     |

**Response:** Created principle object (201)

**Errors:** 400 if title is missing or empty


==== Get Principle ====

**GET /api/principles/<principle_id>**

Returns a specific principle by ID.

**Response:** Principle object, or 404 if not found


==== Update Principle ====

**PUT /api/principles/<principle_id>**

Updates a principle's properties.

| Field       | Type   | Description                               |
|-------------|--------|-------------------------------------------|
| title       | string | Principle statement (cannot be empty if provided) |
| description | text   | Elaboration text                          |
| color       | string | Hex color code                            |

**Response:** Updated principle object, or 404 if not found

**Errors:** 400 if title is set to empty


==== Delete Principle ====

**DELETE /api/principles/<principle_id>**

Permanently deletes a principle and removes it from all attached entities.

**Response:** 204 No Content, or 404 if not found


===== Association Endpoints =====

==== Attach Principle to Entity ====

**POST /api/principles/attach**

Attaches a principle to an entity.

| Field        | Type    | Required | Description                              |
|--------------|---------|----------|------------------------------------------|
| principle_id | integer | yes      | ID of the principle to attach            |
| target_type  | string  | yes      | Entity type (see principled entities)    |
| target_id    | integer | yes      | ID of the target entity                  |

**Business Logic:**
* Idempotent: attaching the same principle twice has no effect
* Entity must support principles (have PrincipledMixin)

**Response:** { "success": true } (200)

**Errors:**
* 400 if required fields missing
* 404 if principle or target entity not found


==== Detach Principle from Entity ====

**POST /api/principles/detach**

Removes a principle from an entity.

| Field        | Type    | Required | Description                              |
|--------------|---------|----------|------------------------------------------|
| principle_id | integer | yes      | ID of the principle to detach            |
| target_type  | string  | yes      | Entity type (see principled entities)    |
| target_id    | integer | yes      | ID of the target entity                  |

**Business Logic:**
* Idempotent: detaching a non-attached principle has no effect

**Response:** { "success": true } (200)

**Errors:**
* 400 if required fields missing
* 404 if principle or target entity not found


===== Principled Entities =====

| Entity Type | Description                |
|-------------|----------------------------|
| task        | A task (standalone or subtask) |
| project     | A project                  |
| routine     | A routine template         |
| session     | A specific session occurrence |
| calendar    | A calendar container       |


===== Principle Object =====

| Field       | Type     | Description                         |
|-------------|----------|-------------------------------------|
| id          | integer  | Primary key                         |
| user_id     | integer  | Owner                               |
| title       | string   | Principle statement                 |
| description | text     | Elaboration on the principle        |
| color       | string   | Hex color code                      |
| created_at  | datetime | When created                        |
| updated_at  | datetime | When last modified                  |


===== Design Philosophy =====

**Why separate Tags and Principles?**

* **Prevents clutter**: Principles won't get buried among dozens of organizational tags
* **Encourages intentionality**: Having a dedicated "principles" concept prompts reflection on values
* **Enables different UI treatment**: Principles can be displayed prominently; tags can be compact
* **Supports different queries**: "Show me everything tied to this principle" vs "filter by these tags"

**Intended Usage Pattern**
* Tags: Create freely as needed, use for filtering and organization
* Principles: Define thoughtfully (maybe 5-10 total), attach to reinforce motivation and alignment
