====== Commitments ======
Created Sunday 23 March 2026


===== Priority 1: Smoke Tests (Critical Path) =====

These validate that basic commitment functionality works at all.

==== ✅️Smoke Test List ====
1. **Create a soft commitment with timeframe_id** - POST /api/commitments/soft with task, timeframe_id
2. **Create a hard commitment** - POST /api/commitments/hard with task, due_at
3. **List commitments** - GET /api/commitments (verify commitment appears)
4. **Get commitment by ID** - GET /api/commitments/{id}
5. **Update commitment status** - PATCH /api/commitments/{id} with status
6. **Delete commitment** - DELETE /api/commitments/{id}

===== Priority 2: Business Logic Tests =====

==== Soft vs Hard Commitment Behavior ====
87. **Soft commitment has no due_at or start_at** - Verify timestamps remain null
88. **Hard commitment has due_at populated** - Verify due_at is present
89. **Hard commitment may have start_at** - Verify start_at can be set
90. **Hard commitment auto-creates day timeframe** - Verify correct day derived from due_at

==== Duplicate Prevention ====
91. **Cannot commit same target to same timeframe twice (soft)** - Verify 409
92. **Cannot commit same target to same day twice (hard)** - Verify 409
93. **Can commit same target to different timeframes** - Verify success
94. **Can commit different targets to same timeframe** - Verify success

==== Timezone Handling ====
95. **List commitments converts UTC to local time** - Verify due_at/start_at in response are local
96. **Get commitment converts UTC to local time** - Verify due_at/start_at in response are local
97. **Create hard commitment accepts local time** - Verify due_at input is treated as local
98. **Update commitment accepts local time** - Verify due_at/start_at inputs are treated as local

==== Status Values ====
99. **Commitment status defaults to planned** - Verify default on creation
100. **Commitment status can be any valid value** - Verify all status values accepted

==== Target Type Restrictions ====
101. **Projects cannot receive hard commitments** - Verify 400 error
102. **Projects can receive soft commitments** - Verify success
103. **Tasks can receive both soft and hard** - Verify success for both
104. **Routines can receive both soft and hard** - Verify success for both
105. **Sessions can receive both soft and hard** - Verify success for both
