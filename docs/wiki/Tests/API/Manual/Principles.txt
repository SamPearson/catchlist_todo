====== Principles ======
Created Wednesday 08 April 2026

===== Priority 1: Smoke Tests (Critical Path) =====

These validate that basic principle functionality works at all.

==== Smoke Test List ====
1. **Create a basic principle** - POST /api/principles with only title
2. **List principles** - GET /api/principles (verify principle appears)
3. **Get principle by ID** - GET /api/principles/{id}
4. **Update principle title** - PUT /api/principles/{id}
5. **Delete principle** - DELETE /api/principles/{id}


===== Priority 2: Extended CRUD Tests =====

==== List Principles (GET /api/principles) ====
6. **List principles returns empty array** - New user with no principles gets []
7. **List principles only returns user's own principles** - Create principles for two different users, verify isolation
8. **List principles returns all principles** - Create 5 principles, verify all returned (no filtering)

==== Create Principle (POST /api/principles) ====
9. **Create principle with all fields** - title, description, reason, and color
10. **Create principle verifies defaults** - Verify color defaults to #ffd700 (gold) when not specified
11. **Create principle with custom color** - Verify color is set correctly
12. **Create principle with description only** - Verify title + description works
13. **Create principle with reason only** - Verify title + reason works
14. **Create principle with description and reason** - Verify both fields stored
15. **Create principle without title** - Verify 400 error
16. **Create principle with empty title** - Verify 400 error
17. **Create principle with whitespace-only title** - Verify 400 error
18. **Create principle with title exactly 100 chars** - Verify success (boundary)
19. **Create principle with title over 100 chars** - Verify 400 error (boundary)
20. **Create principle with long description** - Verify text field accepts long content
21. **Create principle with long reason** - Verify text field accepts long content
22. **Create principle with empty request body** - Verify 400 error
23. **Create duplicate principle title** - Verify allowed (principles can have same title)

==== Get Principle (GET /api/principles/{id}) ====
24. **Get principle returns full object** - Verify all fields present (id, user_id, title, description, reason, color, created_at, updated_at)
25. **Get nonexistent principle** - Verify 404
26. **Get another user's principle** - Verify 404

==== Update Principle (PUT /api/principles/{id}) ====
27. **Update principle title only** - Verify title changes, other fields unchanged
28. **Update principle description only** - Verify description changes, other fields unchanged
29. **Update principle reason only** - Verify reason changes, other fields unchanged
30. **Update principle color only** - Verify color changes, other fields unchanged
31. **Update principle title and description together** - Verify both change
32. **Update principle all fields together** - Verify title, description, reason, color all change
33. **Update principle with empty title** - Verify 400 error
34. **Update principle with whitespace-only title** - Verify 400 error
35. **Update principle clear description with empty string** - Verify description can be cleared
36. **Update principle clear reason with empty string** - Verify reason can be cleared
37. **Update principle with no data** - Verify 400 error or accepts (check API behavior)
38. **Update principle with empty request body** - Verify 400 error
39. **Update nonexistent principle** - Verify 404
40. **Update another user's principle** - Verify 404

==== Delete Principle (DELETE /api/principles/{id}) ====
41. **Delete principle returns 204** - Verify response
42. **Delete principle actually removes it** - Verify GET returns 404 afterward
43. **Delete principle with attachments (cascade)** - Attach principle to task, delete principle, verify association removed
44. **Delete nonexistent principle** - Verify 404
45. **Delete another user's principle** - Verify 404
